메인 담당 (Android/iOS SDK, App) · 라온시큐어

금융·공공기관에 납품하는 모바일 보안 SDK. 다양한 단말 환경에서의 안정성이 핵심인 제품에서, 코드 구조 전환과 성능 안정화를 주도했다.

아키텍처 설계: Java→Kotlin 전환을 설계하여 코드 가독성·유지보수성 향상. 전체 아키텍처를 MVVM으로 리팩토링하여 테스트 가능성과 확장성 확보.
  • 성능 최적화: WorkManager·Coroutines 기반 비동기·병렬 처리 설계. 초기 빈번했던 ANR과 메모리 릭을 JNI 레벨(GlobalRef 릭, native allocation 미해제)까지 추적·해결하여 안정화.
  • 네이티브 모듈: NDK/JNI 기반 시스템 레벨 모듈 설계·구현.
  • 품질 관리: Unit Test 기반 SDK 품질 관리. Jenkins CI/CD를 통한 빌드·배포 자동화. Firebase 도입으로 원격 로그 수집·분석 체계 구축.
  • 탐지 → 대응 흐름: 악성앱/보이스피싱앱 탐지 → 위험 앱 리스트 노출 → 사용자가 직접 삭제를 선택하는 end-to-end 흐름 설계.
  • 보안 기능: Magisk/RootHide 등 진화하는 루팅 기법 분석 및 탐지 기능 설계. 시스템 보안 정책(SandBox) 경계를 분석하여 원격 탐지 기법 설계. Frida 기반 우회 테스트로 탐지 로직 검증. FakeCall, BankingTrojan 등 공격 PoC 구현으로 방어 로직 사전 검증.
  • SDK 호환성: Android 8~15, 주요 제조사(삼성·LG·샤오미 등) 대응. Reflection 활용으로 제조사·OS별 비공개 API 차이를 흡수하는 호환성 레이어 설계. 금융·공공기관 다수 납품. 플레이스토어 배포·업데이트 운영.

AI 보이스피싱 탐지 — FP Rate 분석 기반 제품화 판단

2025.09 — 2025.12
전체/모델 개발 리드 (3인 팀) · 라온시큐어

보이스피싱 통화를 AI로 실시간 탐지하는 연구. Recall 99.34% 달성했으나, 높은 Recall 자체가 과적합 시그널이었고 실서비스 품질에 미달하여 제품화를 보류했다.

판단 근거: Recall 99.34%는 현업 기준에서도 과적합으로 판단. FP Rate 22%(정상 통화 약 5건 중 1건 오탐지). 실제 사용 시 체감 오탐률이 수치보다 높아 사용자 신뢰를 훼손할 수준으로 분석. 출시를 보류하고 후속 개선 전략(Hard Negative Mining, 정상 대화 합성 데이터 보강)을 수립.
  • KoBERT 기반 모델 아키텍처 설계 및 학습 파이프라인 구축.
  • "Recall이 높으면 출시"가 아닌, 과적합 여부와 체감 품질까지 검증하여 Go/No-Go를 판단한 경험.

AI 딥페이크 탐지 — 온디바이스 vs 클라우드 아키텍처 전환 설계

2024.04 — 2024.12
앱 개발 리드 / 모델 개발 서브 (6인 팀) · 라온시큐어

오픈소스 딥페이크 탐지 모델을 모바일 보안 서비스에 응용. 앱 개발을 메인으로, 모델 튜닝을 서브로 담당하며 아키텍처 전환을 설계.

설계 판단: 모바일 환경에서의 온디바이스 실시간 탐지를 위해 모델 양자화(FP32→INT8) 실험 수행. 추론 속도 및 MediaProjection 캡처 해상도 제약(최대 720p)을 정량 측정한 뒤, 실시간 영상통화(30fps) 대비 처리율 부족으로 판단하여 클라우드 비동기 아키텍처로 전환 설계 — 서비스 출시 일정 준수.
  • 서비스 흐름: 사용자가 영상 업로드 또는 URL(유튜브/SNS) 입력 → 클라우드 탐지 → 딥페이크 확률(%) 결과 제공.
  • 기존 모델의 동양인 탐지율 한계 식별, 자체 데이터셋 구축으로 정확도 95% 달성.
  • 50만 다운로드 B2C 앱(라온 모바일 시큐리티)에 탑재하여 서비스 출시.
